claude‑design‑export

Convert Claude Design HTML animations to high-quality MP4 video. Open source, one command, zero system dependencies.

Install View on GitHub

Demo: a 15-second Claude Design film, exported at 1080p60 with 2× supersampling.

Why

Claude Design generates polished animated HTML "micro-films", but there's no built-in way to save them as video. This tool does that — deterministically and at high quality.

  • Frame-accurate. Drives the internal timeline directly instead of recording wall-clock playback.
  • High quality. 2× supersampling by default; BT.709 color; CRF 15 near-lossless H.264.
  • Zero setup. npm install bundles Chromium and ffmpeg. Nothing else required.
  • No bundle edits. Injection is browser-side only; the HTML file is never modified.

Install

git clone https://github.com/Lolarz20/claudetovideo.git
cd claudetovideo
npm install

Usage

node bin/cli.js film.html
# → film.mp4

Defaults are tuned for highest quality: 60 fps, 2× supersampling, CRF 15, libx264 slow preset.

Quick preview (≈ 3× faster)

node bin/cli.js film.html --fast

Archival lossless master

node bin/cli.js film.html --crf 0 --ss 2 --preset veryslow

All options

-o, --output <path>    Output MP4 path (default: <input>.mp4)
--fps <n>              Frame rate (default: 60)
--ss <n>               Supersample factor (default: 2, 1 disables)
--crf <n>              libx264 CRF, 0=lossless .. 51 (default 15)
--preset <name>        ultrafast .. veryslow (default slow)
--fast                 Shortcut: --ss 1 --crf 20 --preset medium
--headed               Show browser window (debug)
--verbose              Stream ffmpeg logs

How it works

Claude Design bundles React plus a custom Stage component that owns a timeline in React state ({ time, duration, playing, setTime, setPlaying }). setTime is stable across renders but lives inside a closure, so it isn't reachable from outside.

  1. Load the HTML in headless Chromium via Playwright at deviceScaleFactor = supersample.
  2. Before any page script runs, define a getter/setter on window.React that traps the moment the UMD factory populates React.createElement. Every element call is sniffed — when props match the TimelineContext value shape, setTime and setPlaying are stashed on window.__stage.
  3. Read Stage dimensions off the Stage component's own props the same way.
  4. Resize the viewport so Stage's auto-fit scale is exactly 1:1.
  5. Freeze CSS animations (animation-play-state: paused) so non-timeline animations stay deterministic.
  6. For each frame: setTime(i / fps), wait two requestAnimationFrames for React to commit + paint, screenshot clipped to the stage.
  7. With supersample > 1, ffmpeg downsamples PNGs with Lanczos before encoding — classic SSAA for crisp fonts and thin lines.
  8. Pipe PNGs through libx264 with CRF 15, BT.709 color metadata, and +faststart.

Limitations

  • Only works with Claude Design's __bundler/* HTML format (as of 2026-04).
  • Only films that use the Stage / TimelineContext timeline component.
  • No audio export — Claude Design doesn't produce audio today.
  • Output is MP4 (H.264) only. WebM / GIF / PNG sequence planned.
